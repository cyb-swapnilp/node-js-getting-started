Change Orders & Exception Handling

You need to change the output spec for the digital master: 

    For assets already processed, do we transcode those assets internally, or have DMS encode them directly from the original source again?  What if the source isn't DMS?

 

    Do we re-run any of the machine or human analysis steps?  Is it possible that something like timing of subtitles, or other analysis results, would be impacted the change?

 

    Do we use the same work order?

You need to add a proxy:

    For assets already processed, do we derive the new proxy ourselves, or does DMS do that?

 

    Do we use the same work order?

Is there a use case for persistent work orders? 

    We define a process flow to use over an indeterminate time for an unknown number of assets.  Do we create a work order and leave it open for any new assets that come in whenever they come in?  Or do we create a new work order from a saved template for each new batch of assets?

QC passes a Digital Master but fails an associated proxy:

    Do we recreate the proxy ourselves, or have DMS do it?

Service removal 

    We decide half way through a work order to remove facial recognition or another individual service from the work order.  Do we discard the data we've already gathered related to that service?  Do we need to decide this on a case by case basis?
